최근 커밋 내용을 기반으로 기술 블로그 포스트를 작성합니다.

## 작성 스타일

### 문체
- 존댓말 사용 (~입니다, ~합니다)
- 전문적이고 깊이 있는 설명
- 시니어 개발자 관점에서 WHY에 대한 설명 포함

### 문서 구조
```
# [작업 제목]

<br>

## 개요

[작업 요약 - 무엇을 했고, 왜 했는지 1-2문장]

<br><br>

## 1. 문제 분석

### 1.1 [문제 정의]
[문제에 대한 설명]
<br>

### 1.2 기존 코드의 문제점
[기존 코드와 문제점 분석]

<br><br>

## 2. 해결 전략

### 2.1 접근 방식 선택
[여러 방법 비교표 + 선택 이유]
<br>

### 2.2 개선된 코드
[개선된 코드와 설명]
<br>

### 2.3 [추가 구현 사항]
[필요시 추가]

<br><br>

## 3. [추가 작업이 있다면]

[테스트, 인프라 구축 등]

<br><br>

## 4. 성능 비교 (해당시)

| 지표 | Before | After |
|------|--------|-------|
| ... | ... | ... |

<br><br>

## 5. 결론

[핵심 내용 3줄 요약]

<br><br>

## 변경 파일 목록

| 파일 | 변경 내용 |
|------|-----------|
| ... | ... |
```

### 간격 규칙
- `# 제목`과 `## 개요` 사이: `<br>`
- `## 섹션` 사이: `<br><br>`
- `### 소제목` 사이: `<br>`
- `## 제목`과 첫 `###` 사이: `<br>` 없음

### 스타일 규칙
- 코드 블록 상단에 주석으로 파일명 표시: `// FileName.java`
- **굵은 글씨**로 핵심 포인트 강조
- 표를 활용한 비교 (방법 비교, 성능 비교)
- 번호 목록으로 포인트 정리
- 설계 의도와 트레이드오프 설명 포함

## 실행

1. `git show` 또는 `git log`로 최근 커밋 내용 확인
2. 변경된 파일들의 diff 분석
3. 위 구조에 맞춰 `docs/YYYY-MM-DD-[slug].md` 파일 생성
